为什么 Sobel 算子是这个形式？

无业游民
人民艺术家

之前只知道 Sobel 算子可以用于检测边缘，无意中看到有人用它近似计算图像横、纵向梯度。遂对各种计算图像梯度的算子产生了好奇，为啥它们有各种不同的形式？

谷歌了下，发现 stack overflow 上早有人提过了这个问题：

Why Sobel operator looks that way? 高赞答案说的非常好，这里翻译一下。

以下为原答案：

在计算机视觉领域，不存在完美、普适的方法做某件事。通常情况下，我们只是随便选一个算子，看它的结果是否符合需求。这个道理也适用于计算图像的梯度：Sobel 算子只是多种梯度算子之一，它在很多场景下被证明很有用。

最简单的梯度算子是下面这种形式的：

[-1, 1]

这个算子虽然简单，但是它存在第一个缺陷：当你用它时，你搞不清楚自己到底在计算哪个位置的梯度。比如当你把它施加在两个相邻的像素：(x, y) 和 (x+1, y) 上时，你到底计算的是 (x, y) 处的梯度，还是 (x+1, y) 处的梯度呢？事实上，你计算的是 (x+0.5, y) 处的梯度，但计算半个像素上的梯度太诡异了。


这就是为啥要在中间加个0：

[-1, 0, 1]

把上面的算子施加到三个相邻的像素上：(x-1, y)，(x, y) 和 (x+1, y) 上，会明确得到 (x, y) 处的梯度，不再产生歧义。

[-1, 0, 1] 其实可以看成两个[-1, 1] 算子组合的产物：

[-1, 1, 0] 计算 (x-0.5, y) 处的梯度，即目标像素左边的梯度，而 [0, -1, 1] 计算 (x+0.5, y) 处的梯度，即目标像素右边的梯度。再加起来，便是目标像素处的梯度。

但这个算子还有缺陷：它对噪声太敏感了。这就是为什么我们决定不只把它施加到一行，而是三行像素上，下面这样就能降低部分噪声了：

[-1, 0, 1]
[-1, 0, 1]
[-1, 0, 1]

但是这么做又过于平均主义了：当把算子施加到某一个特定行时，会导致失去该行像素的细节。为了解决这个问题，我们希望给中间行的像素分配大一点的权重，这样就能在降低噪音的同时，保持中间行的特殊性。这样就得到了 Sobel 算子：

[-1 0 1]
[-2 0 2]
[-1 0 1]

改改系数又能得到其它的梯度算子，例如 Scharr 算子，仅仅是给中心行分配了更多的权重：

[-3  0 3 ]
[-10 0 10]
[-3  0 3 ]

这么做也有一些数学上的原因，例如滤波器的 separability……但是我更喜欢把它看成是一个实验发现，它恰好被证明具备某些有趣的数学特性。在我看来，实验是计算机视觉的核心，单靠想象很难创造出新的算子，必须通过实验证明算子有效才行。
